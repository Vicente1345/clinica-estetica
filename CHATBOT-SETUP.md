# Setup del Chatbot IA con Solicitudes de Pacientes

El chatbot usa **Claude (Anthropic)** vía una **Vercel Serverless Function** y puede registrar solicitudes de pacientes directamente en Supabase. Para que funcione necesitas 4 cosas:

---

## 1. Crear la tabla en Supabase

Ve al panel de Supabase → SQL Editor → pega y ejecuta este SQL:

```sql
create table solicitudes_paciente (
  id bigint generated by default as identity primary key,
  created_at timestamptz default now(),
  nombre text not null,
  telefono text not null,
  email text,
  rut text,
  motivo_consulta text not null,
  especialidad_preferida text,
  profesional_preferido text,
  fecha_preferida text,
  notas_adicionales text,
  estado text default 'pendiente',
  admin_obs text,
  contactado_por text,
  contactado_at timestamptz
);

-- Permitir que el chatbot (anon) inserte solicitudes
alter table solicitudes_paciente enable row level security;

create policy "Permitir insert anon" on solicitudes_paciente
  for insert to anon with check (true);

create policy "Permitir lectura/edición autenticados" on solicitudes_paciente
  for all to authenticated using (true) with check (true);

-- Si tu app usa la anon key incluso después del login (como es el caso),
-- también permite lectura/edición desde anon:
create policy "Permitir todo anon" on solicitudes_paciente
  for all to anon using (true) with check (true);
```

> ⚠️ La política "todo anon" es porque tu app usa la anon key para todo (no usa Supabase Auth). Si más adelante migras a Supabase Auth, ajusta las policies.

---

## 1.5. Migración Sprint 3A — Disponibilidad real-time

Si ya tenías la tabla creada, ejecuta este SQL adicional para agregar las columnas de fecha/hora/box:

```sql
alter table solicitudes_paciente
  add column if not exists fecha_solicitada date,
  add column if not exists hora_inicio time,
  add column if not exists hora_fin time,
  add column if not exists box_tipo text,
  add column if not exists tratamiento text;

-- Índice para búsquedas de disponibilidad rápidas
create index if not exists idx_solicitudes_disp
  on solicitudes_paciente (fecha_solicitada, box_tipo, estado);
```

> Estados válidos: `pendiente` (sin fecha), `agendada` (con fecha+hora bloqueada), `contactado`, `confirmada`, `descartada`. Las que cuentan como "ocupado" para disponibilidad: `agendada`, `contactado`, `confirmada`.

## 2. Instalar dependencias

En la terminal del proyecto (VS Code):

```bash
npm install
```

Esto instala `@anthropic-ai/sdk` que se agregó a `package.json`.

---

## 3. Generar API Key de Anthropic

1. Entra a https://console.anthropic.com
2. Crea cuenta o inicia sesión
3. Settings → API Keys → Create Key
4. Copia la key (empieza con `sk-ant-...`) — **no se muestra de nuevo**
5. Agrega saldo en Settings → Billing (mínimo $5 USD para empezar)

---

## 4. Configurar variables de entorno en Vercel

1. Ve al dashboard de Vercel → tu proyecto `clinica-estetica`
2. Settings → Environment Variables
3. Agrega estas 3 variables (las 2 últimas ya deberían existir):

| Variable | Valor | Notas |
|---|---|---|
| `ANTHROPIC_API_KEY` | `sk-ant-...` | tu key recién generada |
| `REACT_APP_SUPABASE_URL` | `https://xxx.supabase.co` | ya configurada para el frontend |
| `REACT_APP_SUPABASE_ANON_KEY` | `eyJh...` | ya configurada para el frontend |
| `RESEND_API_KEY` | `re_...` | (opcional) email de aviso al admin · ver paso 4.5 |
| `ADMIN_NOTIFY_EMAIL` | `vbecerrai@udd.cl` | (opcional, default: vbecerrai@udd.cl) |
| `RESEND_FROM` | `Barcelona Clinic <onboarding@resend.dev>` | (opcional, hasta verificar dominio) |

Marca los 3 entornos (Production, Preview, Development) en cada una.

4. Click "Save"
5. Ve a Deployments → último deployment → "Redeploy" (para que tome las nuevas variables)

---

## 4.5. Email automático con Resend (Sprint 3A)

Cuando un paciente agenda hora, se manda un email al admin (con datos completos) y otro al paciente (confirmación). Si no configuras Resend, el sistema sigue funcionando — solo se omiten los emails.

**Setup**:
1. Crea cuenta gratis en https://resend.com (3.000 emails/mes gratis)
2. Settings → API Keys → Create API Key → copia el `re_...`
3. Agrégala como `RESEND_API_KEY` en Vercel (paso 4)
4. **Importante — dominio**:
   - **Sin dominio verificado**: usa el `from` por defecto `onboarding@resend.dev`. Resend solo dejará enviar a tu propio email verificado de la cuenta. Funciona para que TÚ recibas avisos pero no para mandar a pacientes.
   - **Con dominio verificado** (recomendado): en Resend → Domains → Add → verifica registros DNS → en Vercel cambia `RESEND_FROM` a algo como `Barcelona Clinic <hola@barcelonaclinic.cl>`. Así puedes mandar a cualquier email.
5. Redeploy Vercel para que tome la variable.

**Probar**:
- Agenda una cita por el chatbot
- Revisa el inbox de `vbecerrai@udd.cl` — debería llegar un email con los datos
- Revisa logs en Vercel → Functions → /api/chat si no llega

## Cómo funciona

### Flujo del paciente (chatbot público en landing)

1. Paciente abre la burbuja del chat
2. Escribe "Quiero agendar hora con la dentista"
3. El bot conduce la conversación y junta: nombre, teléfono, motivo (mínimo)
4. Cuando confirma, el bot **invoca la herramienta `registrar_solicitud_paciente`**
5. Se guarda en la tabla `solicitudes_paciente` con estado `pendiente`
6. El bot le dice al paciente: "Solicitud registrada con código #123. Te contactaremos por WhatsApp en máximo 2 horas hábiles."

### Flujo de la admin/recepción

1. Inicia sesión en el sistema
2. Ve un **badge rojo** en la pestaña "Solicitudes pacientes" con el número de pendientes
3. Click → ve la lista
4. Cada solicitud tiene 4 botones:
   - **💬 WhatsApp**: abre WhatsApp con un mensaje predefinido al teléfono del paciente
   - **Marcar contactado**: registra que ya se le escribió
   - **Marcar agendado**: cuando ya se confirmó la cita
   - **Descartar**: si fue spam, paciente desistió, etc.
   - **📝 Notas**: agregar observaciones internas

## Costos estimados

Con Claude Opus 4.7:
- ~$5 USD por millón de tokens de input
- ~$25 USD por millón de tokens de output
- El system prompt (~3K tokens) se **cachea automáticamente** → 90% más barato en consultas repetidas
- Una conversación típica de 3-5 turnos cuesta menos de **$0.05 USD**

Si el volumen mensual supera tu presupuesto, podemos cambiar a `claude-haiku-4-5` (5x más barato) editando `api/chat.js`.

En console.anthropic.com → Settings → Limits puedes poner un tope mensual de gasto.

---

## Mantenimiento

**Actualizar info de precios o servicios** → Editar el `SYSTEM_PROMPT` en `api/chat.js` y hacer push. Vercel redeploya automáticamente.

**Cambiar el saludo inicial** → Editar la constante `SALUDO` en `src/ChatBot.js`.

**Cambiar qué datos pide el bot** → Editar el `input_schema` de la tool `registrar_solicitud_paciente` en `api/chat.js`.

## Troubleshooting

**El bot responde pero no guarda solicitudes:**
- Revisa que la tabla `solicitudes_paciente` exista en Supabase
- Revisa que las RLS policies permitan insert desde anon
- Revisa Vercel → Logs del último deployment para ver errores

**El bot no responde:**
- Revisa que `ANTHROPIC_API_KEY` esté configurada en Vercel
- Revisa que tengas saldo en console.anthropic.com
- Revisa Vercel → Logs

**El badge de "Solicitudes" no aparece:**
- Recarga la página (los datos se cargan al iniciar sesión)
- Revisa que tu rol sea `admin` o `recep` (los profesionales no ven este tab)
